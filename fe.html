My first taste of front end work came when i was in college, I worked on wordpess sites for local companies including including a gritting one where I added new slideshows and images to their site. Whilst also sprucing up the site template and colours. The Site. I think it may of been changed in recent times though...

When I started my apprenticeship I was extremely green when it came to the front end of web development, my college course had focused on C# and VB.Net. But I was thrown in the deep end when I started work at MrSite, a company where I was working on a moder website building tool written in the modern Angular JS framework. I started of working mostly in the back end and database side off the application but little by little I was eased into using HTML, CSS and JavaScript by the other devs. It started with fixing sites or the preset templates HTML and CSS issues, but eventually grew where i was able to write some back end code to manipulate and work with data and objects for the sites, and then create the front end side of the functionallity as well in the modern Angular code. I would also help design sites for clients, wether this was a migration onto our builder or a complete new site.

The work at MrSite definitely helped me grow and become more confident with my front end skills, but it wasn't until I moved to Justis Publishing that I really settled into working the front end off a large web application. Here a huge searchable database of law documents is paired with a modern and simplistic Javascript Application where user experience and clean content is a first. The app was written largely in TypeScript which helped me alot because of my large C# background and I was working with talented Front End and Back End developers who would guide me on the right path. Here I have really brushed up my JavaScript skills so it has become second nature, whilst also understanding the importance of design and UX in websites and apps.

I'm always looking to expand my knowledge and I'm a fast learner, I'm current investigating HTML6 and the possibilities with frameworks like Angular2 and Aurelia.
